---
title: AI Recommended Assets
description: Data-driven ranking + AI interpretation for Bithumb KRW market assets.
openapi: openapi.json GET /v1/market/ai-recommended-assets
---

<div style={{ maxWidth: 760 }}>

<h2 style={{ fontSize: "22px", marginBottom: 8 }}>Overview</h2>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  The <strong>AI Asset Recommendation</strong> API delivers ranked insights for crypto assets listed in the
  <strong>Bithumb KRW</strong> market. It combines multi-dimensional scoring with AI-generated interpretation.
  The response shape depends on the <code>type</code> you request.
</p>

<Note>
  This endpoint returns a point-in-time snapshot. If date is omitted, the most recent snapshot is returned.
</Note>

<h3 style={{ marginTop: 20 }}>Response modes (type)</h3>

<ul style={{ fontSize: 15, lineHeight: 1.8 }}>
  <li>
    <strong>score_ranking</strong>: asset info + <code>total_score</code>, <code>score_by_category</code>,
    <code>overall_rank</code>, and <code>ranking_by_category</code>.
  </li>
  <li>
    <strong>summary</strong>: asset info + <code>summary</code> (AI-generated one-liner).
  </li>
  <li>
    <strong>comprehensive_evaluation</strong>: asset info + <code>total_score</code>, <code>score_by_category</code>,
    <code>overall_rank</code>, <code>opinion</code>, and <code>interpretation</code>.
  </li>
  <li>
    <strong>sentiment_trend</strong>: asset info + <code>sns_sentiment</code> distribution (bullish/bearish).
  </li>
</ul>

<p style={{ fontSize: 15, lineHeight: 1.8 }}>
  Each asset item always includes <code>id</code> and asset fields: <code>symbol</code>, <code>name</code>, <code>image_url</code>.
</p>

<h3 style={{ marginTop: 20 }}>Scoring model</h3>

<div style={{ fontSize: 15, lineHeight: 1.8 }}>
  <p style={{ marginBottom: 8 }}>
    Total Score is derived from five categories (each 0-10):
  </p>

  <ul>
    <li><strong>Market Sentiment</strong></li>
    <li><strong>Price Stability</strong></li>
    <li><strong>Search Trend</strong></li>
    <li><strong>Trading Activity</strong></li>
    <li><strong>Upside Energy</strong></li>
  </ul>

  <p style={{ marginTop: 10 }}>
    The response includes <code>score_by_category</code> and, for comprehensive evaluations,
    <code>interpretation</code> labels per category.
  </p>
</div>

<h3 style={{ marginTop: 20 }}>How to use</h3>

<Steps>
  <Step title="Provide tickers">
    Pass <code>tickers</code> as a comma-separated list (required). Example: <code>BTC,ETH,SOL</code>.
  </Step>

  <Step title="Choose a response type">
    Set <code>type</code> (required) to one of: <code>score_ranking</code>, <code>summary</code>,
    <code>comprehensive_evaluation</code>, <code>sentiment_trend</code>.
  </Step>

  <Step title="Optional: request a historical snapshot">
    Provide <code>date</code> (YYYY-MM-DD). The backend treats it as KST 00:00 for that day and returns
    the most recent data if omitted.
  </Step>

  <Step title="Read the snapshot">
    The response includes <code>generated_at</code> (KST) and an <code>assets</code> list for the tickers that exist in
    the backend dataset.
  </Step>
</Steps>

<Warning>
  Unknown tickers are omitted. If certain data is unavailable or not applicable, corresponding fields may be
  <code>null</code> or omitted.
</Warning>

</div>
